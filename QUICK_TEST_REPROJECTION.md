# 快速测试: Gimbal Yaw 重投影修复

## 🎯 测试目标
验证修改 `gimbal_yaw` 参数是否会实时影响重投影结果(绿色装甲板框)。

## 🚀 测试步骤

### 1. 运行程序
```bash
cd /home/prototype152/桌面/EX_MiracleVision/build
LD_LIBRARY_PATH=./lib:../3rdparty/foxglove/lib ./bin/minimum_vision
```

### 2. 在 Foxglove Studio 中连接
1. 打开 https://app.foxglove.dev
2. 连接到 `ws://192.168.80.128:8765`
3. 添加以下面板:
   - **Image Panel** - 订阅 `/video_feed`
   - **Parameters Panel** - 显示所有参数

### 3. 测试参数修改

#### 测试 A: 暂停模式测试 (推荐)
1. **暂停视频**: 在程序窗口按空格键
2. **修改参数**: 在 Foxglove Parameters 面板中:
   - 找到 `gimbal_yaw`
   - 当前值: `0.0`
   - 修改为: `10.0` (顺时针旋转 10 度)
3. **继续播放**: 按任意键播放一帧
4. **观察结果**:
   - ✅ 绿色装甲板框应该向右旋转
   - ✅ 终端显示: `[Param] Updated gimbal_yaw: 10.0`

#### 测试 B: 连续修改测试
1. **持续播放**: 不暂停视频
2. **快速修改**: 
   - `gimbal_yaw`: 0 → 5 → 10 → 15 → 20
   - `gimbal_pitch`: 0 → 5 → 10
3. **观察结果**:
   - ✅ 绿色框应该实时跟随参数变化
   - ✅ Yaw 增加 → 框向右旋转
   - ✅ Pitch 增加 → 框向下移动

## 🐛 故障排除

### 问题 1: 修改参数后没有变化
**可能原因**:
- 视频帧率太快,变化不明显
- 追踪器状态为 `lost`,没有目标

**解决方法**:
- 暂停视频后修改
- 确保有装甲板被追踪

### 问题 2: 程序崩溃
**可能原因**:
- 死锁问题未完全解决

**解决方法**:
- 检查终端错误信息
- 确保使用最新编译的版本

### 问题 3: 终端没有显示 "[Param] Updated..."
**可能原因**:
- 参数回调未触发
- 参数名称不匹配

**解决方法**:
- 检查 Foxglove 中参数名是否为 `gimbal_yaw`
- 查看终端是否有其他错误

## 🎓 技术说明

**为什么现在能工作了?**

修复前,重投影使用的是**帧开始时**的云台姿态快照:
```cpp
// 帧开始
current_mock_data = mock_data;  // 快照
solver.set_R_gimbal2world(...);  // 设置一次

// ... 很多代码 ...

// 重投影 (使用旧值)
solver.reproject_armor(...);
```

修复后,重投影前**重新获取最新值**:
```cpp
// 帧开始
current_mock_data = mock_data;
solver.set_R_gimbal2world(...);

// ... 很多代码 ...

// 重投影前重新获取
realtime_mock_data = mock_data;  // 最新值
solver.set_R_gimbal2world(...);  // 重新设置

// 重投影 (使用新值) ✅
solver.reproject_armor(...);
```
